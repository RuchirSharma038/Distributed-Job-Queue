Actions
Dead Letter Queue

Reset all dead jobs back to queued with a fresh retry budget. Use this after an outage to recover failed jobs at once.

Zombie Hunter

Scan for jobs stuck in running state for longer than the timeout. These are workers that were killed by the OS mid-job. Sends them back into the retry loop.

Default timeout: 15 min
Dual-Write Reconciliation

Finds jobs where the Postgres write succeeded but the matching Redis write never happened — these are invisible to the zombie hunter because their status never reached running. Safe to run anytime; re-pushing an already-queued job is a no-op for workers.
